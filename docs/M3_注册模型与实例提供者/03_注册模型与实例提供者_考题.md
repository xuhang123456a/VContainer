# M3 注册模型与实例提供者 · 考题

## 🟢 概念题

1. `Registration` 里有没有保存"单例实例"？单例实例到底缓存在哪？
2. 同一接口 `IFoo` 注册了 3 个实现，`Resolve<IFoo>()` 返回哪个？`Resolve<IEnumerable<IFoo>>()` 返回什么？
3. `IInstanceProvider` 的 6 种实现各对应什么注册方式？
4. `OpenGenericInstanceProvider.SpawnInstance` 被调用会发生什么？为什么？
5. `RegistrationBuilder.As(typeof(IBar))` 在 `Impl` 没实现 `IBar` 时何时报错？

<details><summary>参考答案要点</summary>

1. 没有。`Registration` 全 `readonly`、无实例状态。单例实例缓存在容器侧的 `sharedInstances`（M4），所以同一 Registration 可被多作用域共享并作字典 key。
2. `Resolve<IFoo>()` 返回最后注册的（后覆盖前）；`Resolve<IEnumerable<IFoo>>()` 返回包含全部 3 个的数组。
3. `InstanceProvider`=Register<T>构造注入、`ExistingInstanceProvider`=RegisterInstance、`FuncInstanceProvider`=Register(Func)、`CollectionInstanceProvider`=自动派生集合、`OpenGenericInstanceProvider`=开放泛型 Register、`ContainerInstanceProvider`=IObjectResolver 自身、`ContainerLocalInstanceProvider`=ContainerLocal<T> 回退。
4. 抛 `InvalidOperationException`。它是"注册工厂"不是"实例工厂"，实例由它生产的封闭 Registration 完成。
5. 声明期（`AddInterfaceType` 调用时）即抛 `VContainerException`，不等到解析。
</details>

## 🟡 机制题

1. `Registry.Build` 中 `AnyKey` 这个 sentinel 对象起什么作用？
2. 集合注册是什么时候被自动合成的？它的生命周期是什么，为什么？
3. 封闭泛型解析时，`GetClosedRegistration` 如何避免重复 `MakeGenericType`？缓存键是什么？
4. `Registry.TryGet` 对封闭泛型的三层回退顺序是什么？
5. `RegistrationElement` 这个 struct 携带哪两个信息？为什么集合解析需要它？
6. `Registry` 里为什么会存"value 为 null 的占位项"？

<details><summary>参考答案要点</summary>

1. 追踪"某 service 是否已经有任意成员（不分 key）"。首次注册时同时写 `(svc,key)` 与 `(svc,AnyKey)`；第二次发现 `(svc,AnyKey)` 已存在即触发集合合成。
2. 在 `AddToBuildBuffer` 检测到同 service 第二次注册时合成。集合引用本身是 `Transient`（每次 resolve 新建数组），但成员各自保留自己的生命周期——这样集合不会"锁住"成员的单例性。
3. 用 `TypeParametersKey`(类型参数数组 + Key) 作 `ConcurrentDictionary` 键，`GetOrAdd` 缓存封闭 Registration。
4. ① 开放泛型注册 → 构造封闭；② 单元素集合回退（`IEnumerable<T>`/`IReadOnlyList<T>`）；③ `ContainerLocal<T>` 回退。
5. `Registration` + 注册它的 `IObjectResolver`(容器/作用域)。因为集合成员若是单例，要用"注册它的那个作用域"去 resolve 以保证单例唯一性，瞬时成员才用当前作用域。
6. 当一个注册有 InterfaceTypes 时，其 `ImplementationType` 以 `(Impl,Key)→null` 入表作"存在性标记"，供 `Exists` 检查"impl 是否注册过"，但不作为可解析项（null 占位）。
</details>

## 🔴 架构陷阱题

1. **同底座对比**：`InstanceProvider` 与 `ExistingInstanceProvider` 都产实例，但在「容器 Dispose 跟踪」上待遇不同，差在哪？（联系 M4）
2. 集合解析 `IEnumerable<IFoo>` 时，子作用域注册了 1 个、父作用域注册了 2 个，结果有几个元素？单例成员用谁的作用域解析？
3. 如果忘了在 `AddToBuildBuffer` 里同时更新"单值键"和"集合键"，分别会出现什么 bug？
4. 开放泛型 `Register(typeof(Repo<>)).As(typeof(IRepo<>))`，解析 `IRepo<User>` 的完整链路是怎样的？如果直接让 `OpenGenericInstanceProvider` 产实例会卡在哪？
5. 两个相同 `(Impl, Key)` 的单例被加入同一集合会怎样？为什么 `CollectionInstanceProvider.Add` 有这个检查？
6. `Registration` 不可变且无实例状态，这个设计如何支撑"同一单例在父作用域、子作用域解析得到同一个对象"？
7. 用 `RuntimeHelpers.GetHashCode(type)` 作哈希、`FixedTypeObjectKeyHashtable` 冻结只读，这对"运行时动态新增注册"意味着什么限制？

<details><summary>参考答案要点</summary>

1. `InstanceProvider` 产的实例若实现 `IDisposable` 会被容器加入 `CompositeDisposable` 跟踪释放；`ExistingInstanceProvider`（RegisterInstance）的实例**不被跟踪**（`!(Provider is ExistingInstanceProvider)` 守卫），因为它由外部拥有，容器不该替它 Dispose。
2. 3 个（子 1 + 父 2，`CollectFromParentScopes` 沿父链合并）。单例成员用"注册它的作用域"resolve（`RegistrationElement.RegisteredContainer`），瞬时用当前作用域。
3. 漏单值键覆盖 → `Resolve<IFoo>()` 拿到第一个而非最后注册的；漏集合键 → `Resolve<IEnumerable<IFoo>>()` miss 或只得到单元素回退结果。
4. `TryGet(IRepo<User>)` miss → 是封闭泛型 → 取开放 `IRepo<>` 注册 → 其 Provider 是 `OpenGenericInstanceProvider` → `GetClosedRegistration(IRepo<User>, [User])` 用 `MakeGenericType` 造 `Repo<User>` 的封闭 Registration（带普通 InstanceProvider）→ 容器对封闭 Registration 正常解析。若让开放 Provider 直接 spawn，它不知道要 `User` 这个类型参数（开放 Provider 只持有 `Repo<>`），无法 MakeGenericType。
5. 抛 `Conflict implementation type`。防止把同一个单例实现重复放进集合导致同一对象出现多次（破坏单例语义与集合去重预期）。
6. 因为实例缓存在容器的 `sharedInstances`（key 是 Registration 引用）。子作用域解析单例时会路由到"拥有该注册的根/父容器"去取（见 M4 ResolveCore 的 Singleton 分支），同一 Registration 引用 → 命中同一缓存条目 → 同一对象。
7. 容器构建后注册集**冻结不可变**，运行时无法新增注册（只能 `CreateScope` 建子作用域追加）。这是用"不可变换确定性与 O(1) 查表"的有意取舍。
</details>

## ✍️ 实操题

1. 给 Facade 的 `Registry` 增加「`ContainerLocal<T>` 回退」：解析 `ContainerLocal<T>` 时若 T 已注册，合成一个把 T 包进 `ContainerLocal<T>` 的 Registration。对照 `TryFallbackToContainerLocal`。
2. 实现「单元素集合回退」：`Resolve<IEnumerable<IFoo>>()` 在只注册了 0 或 1 个 `IFoo`（未触发集合派生）时也能返回空数组/单元素数组。对照 `TryFallbackToSingleElementCollection`。
3. 写测试验证「集合引用是 Transient 但单例成员仍是单例」：注册 2 个单例 `IFoo`，两次 `Resolve<IReadOnlyList<IFoo>>()` 得到两个不同数组，但数组里对应元素是同一对象。

<details><summary>参考答案要点</summary>

1. 在 `TryGet` 的泛型回退里加分支：`open == typeof(ContainerLocal<>)` → 取 `T = typeArgs[0]`，`TryGet(T)` 成功则建一个 Provider 在 spawn 时 `new ContainerLocal<T>(resolve(T))`，生命周期 Scoped。
2. 回退里检测 `openType` 是 `IEnumerable<>`/`IReadOnlyList<>` → 取 elementType → 建空 `CollectionInstanceProvider`，若 `(element, AnyKey)` 有单个注册则 Add 进去 → 返回数组型 Registration。
3. 两次 resolve 数组引用 `!=`（Transient 新建），但 `arr1[0] == arr2[0]`（单例成员命中同一 `sharedInstances` 缓存）。
</details>
